一些理论


BM25（Best Matching 25）是一种用于信息检索的排名函数，用于计算查询（Query）与文档（Document）之间的相关性分数。其核心公式如下：

[
\text{score}(D, Q) = \sum_{i=1}^{n} \text{IDF}(q_i) \cdot \frac{f(q_i, D) \cdot (k_1 + 1)}{f(q_i, D) + k_1 \cdot \left(1 - b + b \cdot \frac{|D|}{\text{avgdl}}\right)}
]

公式分项解释：

  1. IDF(q_i)（逆文档频率）

    • 衡量查询词 ( q_i ) 的全局重要性，计算公式通常为：
      [
      \text{IDF}(q_i) = \log \left(\frac{N - n(q_i) + 0.5}{n(q_i) + 0.5} + 1\right)
      ]
      • ( N )：文档集合中的总文档数。
      • ( n(q_i) )：包含词 ( q_i ) 的文档数量。
    • 作用：罕见词（( n(q_i) ) 小）的IDF值更高，对相关性贡献更大。

  2. 词频部分 (\frac{f(q_i, D) \cdot (k_1 + 1)}{f(q_i, D) + k_1 \cdot \left(1 - b + b \cdot \frac{|D|}{\text{avgdl}}\right)})

    • ( f(q_i, D) )：词 ( q_i ) 在文档 ( D ) 中的出现频率（词频）。
    • ( k_1 )（调节参数）：控制词频饱和度的阈值，通常取 ( 1.2 \sim 2.0 )。
      • 当 ( f(q_i, D) ) 增大时，分子增长趋缓，避免高频词过度主导分数。
    • ( b )（长度惩罚系数）：控制文档长度的影响，通常取 ( 0.75 )。
      • ( \frac{|D|}{\text{avgdl}} )：文档 ( D ) 的长度与平均文档长度的比值。
      • 若 ( b = 1 )，长文档会被显著惩罚；若 ( b = 0 )，忽略长度影响。

公式核心思想：

  • 词频与饱和：高频词对分数的贡献非线性增长，避免单个词重复出现导致分数膨胀。
  • 文档长度归一化：长文档因包含更多词可能匹配更多查询词，通过 ( b ) 调整其权重。
  • 罕见词优先：IDF使得罕见匹配词对相关性分数的贡献更大。

参数典型值：

  • ( k_1 = 1.5 ), ( b = 0.75 )（经验值，可根据数据调整）。

示例说明：

假设查询 ( Q = \text{“apple fruit”} )，文档 ( D ) 中：

  • “apple” 出现 3 次（( f=3 )），IDF值为 2.0；
  • “fruit” 出现 1 次（( f=1 )），IDF值为 1.3；
  • 文档长度 ( |D| ) 为平均长度的 1.2 倍（( \frac{|D|}{\text{avgdl}} = 1.2 )），( k_1=1.5 ), ( b=0.75 )。

则分数计算为：
[
\text{score} = 2.0 \cdot \frac{3 \cdot 2.5}{3 + 1.5 \cdot (1 - 0.75 + 0.75 \cdot 1.2)} + 1.3 \cdot \frac{1 \cdot 2.5}{1 + 1.5 \cdot (1 - 0.75 + 0.75 \cdot 1.2)} + \cdots
]

BM25通过平衡词频、文档长度和词稀有性，提供更合理的相关性排序。
